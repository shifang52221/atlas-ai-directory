## Required Fields Gate

Each program row must include all fields below before page-level promotion is allowed:

1. Program URL
2. Application status
3. Payout model
4. Cookie window
5. Payment method
6. Last policy verification date
7. Owner
8. Next action
9. Deadline

If any required field is missing, treat program status as `BLOCKED`.

## Lifecycle States

- `To apply`: not submitted.
- `Applied`: submitted, waiting review.
- `Approved`: program active, links can go live.
- `Blocked`: legal/compliance/payment mismatch, no promotion.
- `Rejected`: denied; move to replacement candidate.

## Weekly Operating Routine

1. Monday: update statuses and blockers.
2. Wednesday: verify terms/cookie/payout changes for approved programs.
3. Friday: reconcile clicks, conversions, and payout data by program.
