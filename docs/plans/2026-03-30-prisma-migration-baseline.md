# Prisma Migration Baseline Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a committed Prisma migration baseline plus repo-level scripts and documentation so existing databases can be safely adopted and future environments can use a standard migration workflow.

**Architecture:** Generate a baseline SQL migration from the current `prisma/schema.prisma`, check in the Prisma migrations directory, and document two supported paths: existing populated databases adopt the baseline via verification plus `migrate resolve`, while new environments initialize from migrations via `migrate deploy`. Keep Prisma pinned to the current major version and verify the repo still lint-tests-builds cleanly after the workflow changes.

**Tech Stack:** Prisma 6.16.2, PostgreSQL, Next.js 16, TypeScript, npm scripts

### Task 2: Create the committed Prisma baseline migration

**Files:**
- Create: `prisma/migrations/20260330_baseline/migration.sql`
- Create: `prisma/migrations/migration_lock.toml`
- Reference: `prisma/schema.prisma`

**Step 1: Create the migrations directory**

Run: `New-Item -ItemType Directory -Force prisma\\migrations\\20260330_baseline`

Expected: PASS with the directory created if it does not already exist.

**Step 2: Generate the baseline SQL from the current schema**

Run:

```powershell
npx prisma migrate diff `
  --from-empty `
  --to-schema-datamodel prisma/schema.prisma `
  --script | Set-Content prisma\migrations\20260330_baseline\migration.sql
```

Expected: PASS with a full SQL baseline written to `prisma/migrations/20260330_baseline/migration.sql`.

**Step 3: Add the migration lock file**

Create `prisma/migrations/migration_lock.toml` with:

```toml
# Please do not edit this file manually
# It should be added in your version-control system (e.g. Git)
provider = "postgresql"
```

**Step 4: Validate Prisma after adding migration artifacts**

Run: `npm run db:validate`

Expected: PASS with no schema validation errors.

**Step 5: Commit**

```bash

Create `docs/runbooks/prisma-migrations.md` with these sections:

```md
# Prisma Migrations Runbook

## Purpose

## Standard Rules
- Do not use `db push` as the standard deployment path
- Do not edit `migration_lock.toml` by hand except to create the initial baseline file
- Do not mark a migration as applied before checking drift

## New Database Setup
```bash
npm run db:up
npm run db:migrate
npm run db:generate
```

## Existing Database Baseline Adoption
1. Back up the database
2. Check drift:
   ```powershell
   npx prisma migrate diff `
     --from-url "$env:DATABASE_URL" `
     --to-schema-datamodel prisma/schema.prisma `
     --exit-code
   ```
3. If drift exists, stop and resolve it before baseline adoption
4. Mark the baseline as applied:
   ```powershell
   npx prisma migrate resolve --applied 20260330_baseline
   ```
5. Verify:
   ```bash
   npm run db:status
   npm run db:generate
   ```

## Future Schema Change Workflow
```bash
npm run db:migrate -- --name <change-name>
npm run db:generate
```

## Staging And Production Deploys
```bash
npm run db:deploy
npm run db:generate
```
```
